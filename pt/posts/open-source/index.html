Open Source

Como escrevi na sessão “Sobre mim”, comecei a me envolver mais na comunidade Open Source. Quero documentar meus estudos em programação e tecnologia em geral, mantendo o princípio KISS, por isso escolhi o Open Source. Não quero entrar na discussão “Free Software Foundation X Open Source Initiative”, pois não é o caso. A ideia é utilizar o modelo Open Source, simplesmente, porque é fácil e atende meus objetivos. Meu objetivo é documentar meus estudos, todavia o Open Source é um ótimo instrumento para mostrar o meu trabalho. Recomendo que todos leiam o Open Source Guides. Este guia está sendo minha fonte de expiração.

Documentar

Existe muito material na internet que fala sobe a importância da documentação em projetos, então vamos começar certo. Conforme o nosso guia, todo projeto deve incluir as seguintes documentações:

  • Open source license
  • README
  • Contributing guidelines
  • Code of conduct

O GitHub, facilita muito este processo, pois é necessário somente salvar esses arquivos no diretório root com os nomes recomendados, que o GitHub irá reconhecê-los e automaticamente mostrá-los da maneira apropriada.

LICENSE

“Todo projeto open source deve ter uma licença open source. Se o projeto não tiver uma licença, não é open source.” Esta afirmativa do nosso guia, nos faz escolher uma licença para o nosso projeto. Quero tratar em post a parte o assunto licença, no site https://choosealicense.com/, há um ótimo material de referencia. Neste projeto irei utilizar a licença MIT, ela é curta e simples. Não posso esquecer do KISS. O GitHub se encarrega de criar o arquivo de licença, quando se inicia um novo repositório, sendo necessário somente escolher o modelo.

Não há desculpas para não adicionar uma licença ao projeto.

README

Explicar um projeto não é algo tão simples, vou usar uma frase do Albert Einstein, para criação dos meus arquivos README, “fazer da forma mais simples possível, mas não mais simples que isso”. Fontes de inspiração, Making READMEs Readable do @18F ou o README template do @PurpleBooth.

CONTRIBUTING

Quero fazer um projeto acolhedor a novas contribuições. Toda e qualquer contribuição será bem vinda. Nunca escrevi um arquivo CONTRIBUTING, vou usar como referência o contributing guide template de @nayafia e o How to Build a CONTRIBUTING.md do @mozilla.

CODE_OF_CONDUCT

Toda contribuição sempre será bem, mas iremos ter regras básicas para o comportamento dos participantes e queremos manter o ambiente amigável e acolhedor. Nosso guia será o Code of Conduct guide.

Então é isso, vou colocar a mão na massa.

Verifique meus projetos no Github. Obrigado por ler!
